# Prompt 10 — Segurança, testes e documentação final

## Prompt

Faz uma revisão final completa do projeto sem adicionar funcionalidades fora do âmbito. Procura SQL injection, XSS, CSRF, controlo de acesso incorreto, palavras-passe expostas, sessões inseguras, alterações via GET, mensagens técnicas públicas, caminhos quebrados e HTML inválido. Corrige os problemas encontrados preservando o comportamento esperado.

Executa verificações de sintaxe PHP e SQL disponíveis no ambiente. Cria um plano de testes manuais que cubra visitante, aluno/professor e administrador, incluindo casos válidos e inválidos. Atualiza a documentação com requisitos, instalação, configuração da base de dados, credenciais apenas de demonstração, estrutura, funcionalidades, limitações e capturas de ecrã ainda necessárias para o relatório da PAP. Não declares como testado aquilo que não foi possível executar.

## Critérios de aceitação

- Não existem vulnerabilidades críticas conhecidas.
- As permissões são testadas para todos os tipos de utilizador.
- Existe um guia de instalação reproduzível.
- Existe uma checklist de testes com resultados registáveis.
- Limitações e verificações não executadas estão explicitamente documentadas.
- O projeto pode ser demonstrado desde a instalação até à moderação de uma ideia.
